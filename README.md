# Tinubu Support Bauchi 2027

A React + Vite single-page application (SPA) for the Tinubu Support Bauchi 2027 citizen movement. It provides public information pages, membership registration, corporate partner registration, news & events, leadership profiles, and role-based admin dashboards.

---

## Tech Stack

- **Framework:** [React 19](https://react.dev/) + [Vite 7](https://vitejs.dev/)
- **Routing:** [TanStack Router](https://tanstack.com/router) with file-based routing
- **State & Data:** [TanStack Query](https://tanstack.com/query) + React Context for auth
- **Styling:** [Tailwind CSS 4](https://tailwindcss.com/) + custom CSS theme tokens
- **UI Components:** Radix UI primitives + shadcn/ui patterns
- **Forms:** React Hook Form + Zod
- **Charts:** Recharts
- **Utilities:** jsPDF, QRCode, date-fns, Lucide icons

### Routing Conventions

Routes are defined by files inside `src/routes/`:

| File | URL |
|------|-----|
| `index.tsx` | `/` |
| `about.tsx` | `/about` |
| `news.tsx` | `/news` |
| `news.$slug.tsx` | `/news/:slug` |
| `verify.$membership.tsx` | `/verify/:membership` |
| `_authenticated/route.tsx` | Layout route guarding authenticated pages |
| `_authenticated/admin.tsx` | `/admin` |
| `_authenticated/dashboard.tsx` | `/dashboard` |

`routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin/vite`. Do not edit it manually.

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) 18+ (or [Bun](https://bun.sh/))
- A running backend API (see `.env.example` for the default URL)

### Installation

```bash
npm install
# or
bun install
```

### Environment Variables

Copy `.env.example` to `.env` and adjust the API base URL:

```bash
cp .env.example .env
```

| Variable | Description | Default |
|----------|-------------|---------|
| `VITE_API_URL` | Base URL of the TSB 2027 REST API | `http://localhost:8080` |

All API calls are made to `${VITE_API_URL}/api/...`.

### Development

```bash
npm run dev
# or
bun dev
```

The dev server runs at `http://localhost:8080` by default.

### Build for Production

```bash
npm run build
# or
bun run build
```

This generates a static `dist/` folder that can be served by any static host (e.g. Nginx, Apache, Vercel, Netlify, Cloudflare Pages).

### Preview Production Build

```bash
npm run preview
```

## Features

- **Public pages:** Home, About, News, Events, Leadership, Contact
- **Membership:** Online registration, member dashboard, membership card generation with QR code
- **Corporate partners:** Corporate registration and certificate download
- **Verification:** Public membership verification by membership number
- **Authentication:** JWT-based login / register with role-based access
- **Admin dashboard:** Member approvals, corporate approvals, messages, subscribers, users, notifications
- **Newsletter:** Subscribe / unsubscribe flows
- **Responsive design:** Mobile-first layout with Tailwind CSS

## API Integration

The API client lives in `src/lib/api.ts`. It wraps `fetch` and expects responses in the envelope:

```json
{ "success": true, "data": { ... } }
```

or on failure:

```json
{ "success": false, "message": "...", "errors": [...] }
```

Authenticated requests automatically attach the JWT stored in `localStorage` under the key `tsb_token`.

### Main API Sections

- `authApi` — login, register, logout, current user, password reset
- `membersApi` — member registration, profile, card download
- `corporateApi` — corporate registration, profile, certificate
- `blogApi` — public news list, categories, single post, admin CRUD
- `eventsApi` — events list, admin CRUD
- `leadershipApi` — leadership list, admin CRUD
- `contactApi` — contact form submissions
- `newsletterApi` — subscribe / unsubscribe
- `notificationsApi` — user notifications
- `adminApi` — admin-only endpoints

## Authentication & Roles

Authentication is handled by `AuthProvider` in `src/lib/auth-context.tsx`. The app stores the JWT in `localStorage` and fetches the current user on mount.

Protected routes are wrapped by `src/routes/_authenticated/route.tsx`, which redirects unauthenticated users to `/auth`.

Supported roles (defined in the backend and typed in `src/lib/api.ts`):

- `super_admin`
- `admin`
- `membership_officer`
- `corporate_officer`
- `editor`
- `member`
- `corporate_user`

## Deployment Notes

This is a **client-side SPA**. After running `npm run build`, serve the contents of the `dist/` folder from any static file host. Make sure your host is configured to fall back to `index.html` for unknown routes so that TanStack Router's client-side routing works correctly.

Example Nginx config snippet:

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```
